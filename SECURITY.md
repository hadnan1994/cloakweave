# Security Policy

## Supported versions

Cloakweave is in early development. Security fixes are handled on the main branch until the first stable release.

## Reporting a vulnerability

Please report security issues privately by opening a GitHub security advisory when available, or by contacting the maintainer directly.

Do not include private documents, secrets, or API keys in public issues.

## Privacy notes

Cloakweave is designed to be local-first:

- Documents are read from your local machine.
- Workspace data is stored locally.
- Embeddings are generated locally by default.
- No telemetry is enabled by default.
- Cloud providers should only be used when explicitly configured.

If you enable a cloud provider, review that provider's data policy before sending private document content.
